01

环境准备

使用你当前 C++ IDE 自带的编译器
说明你现在用的 C++ IDE 已经自带编译器

Dev-C++、Code::Blocks、wxDev-C++ 等 IDE 在安装时都会自带一套 GCC 编译器（一般叫 TDM-GCC 或 MinGW）， 里面包含我们需要的两个工具： 

你将用到的两个命令
g++     // 用来编译 .cpp 源文件
windres // 用来编译 .rc 资源脚本（图标/二进制嵌入等）

以下三种方法按推荐次序任选其一。

方法 A · 推荐使用本目录下的 compile.bat

直接双击项目根目录下的 compile.bat。脚本会自动按正确顺序编译： 

compile.bat 内部流程
[1/4] windres -o app_res.o app.rc          # 主程序图标资源
[2/4] g++    -o app.exe app.cpp app_res.o   # 主程序 exe
[3/4] windres -o inst_res.o installer.rc    # 把 app.exe 作为资源嵌入安装程序
[4/4] g++    -o installer.exe installer.cpp inst_res.o

若双击后出现「g++ 不是内部命令」，请参见方法 B 或下面的「临时添加到 PATH」说明。

方法 B用 Dev-C++ 打开 .dev 工程（GUI 方式）

顺序很重要 —— 因为 Installer.dev 需要 app.exe 已存在才能把它作为资源嵌入。

操作步骤
① 打开 Dev-C++ 菜单： 文件 → 打开项目或文件
② 选择本目录下的 App.dev，点击 运行 → 编译（或按 F9）
③ 编译成功后，再打开 Installer.dev
④ 同样点击 运行 → 编译
⑤ 完成后同目录会出现 app.exe 与 installer.exe
方法 C用 mingw32-make（或 make）编译 Makefile

在 Dev-C++ 菜单： 工具 → 编译选项 → 目录 → Bin 里可以找到编译器 bin 目录路径。

在 cmd 中
set PATH=C:\Dev-Cpp\MinGW64\bin;%PATH%
cd /d "你解压后的项目目录"
mingw32-make   # 或者 make
验证确认编译器是否在 PATH 里

打开命令提示符，输入以下两条命令，若都能输出版本号，表示环境正常：

CMD
g++ --version
windres --version
💡

小提示：若你的 Dev-C++ 安装在 C:\Dev-Cpp， 编译器 bin 目录一般是 C:\Dev-Cpp\MinGW64\bin。 可以临时在当前 cmd 里添加 PATH，再运行 compile.bat： 

CMD
set PATH=C:\Dev-Cpp\MinGW64\bin;%PATH%
compile.bat
02

目录结构与图标准备

放置自定义 app.ico
workspace/ ├── app.cpp // 主程序源文件 ├── app.rc // 主程序资源脚本 ├── installer.cpp // 安装程序源文件 ├── installer.rc // 安装程序资源脚本 ├── manifest.xml // Windows 应用清单 ├── compile.bat // 通用一键编译脚本（推荐） ├── App.dev // Dev-C++ 工程 · 主程序 ├── Installer.dev // Dev-C++ 工程 · 安装程序 ├── Makefile // make / mingw32-make 工程 └── app.ico // ← 你的自定义图标，必须存在
⚠️

图标格式：请使用 .ico 格式（不是 .png/.jpg 直接改后缀）。 推荐含 16×16、32×32、256×256 等多尺寸，以保证任务栏、托盘、快捷方式图标清晰。 推荐在线转换工具：https://convertio.co/png-ico/ 或 https://www.icoconverter.com/。

03

一键构建

双击 compile.bat 完成整个编译流程
Step 1双击 compile.bat

在文件管理器中，直接双击项目根目录下的 compile.bat。

Step 2观察输出

脚本会按顺序执行以下四步，每一步都有中文提示：

CMD 输出示例
[1/4] 编译主程序资源 ...
[2/4] 编译并链接主程序 app.exe ...
[3/4] 编译安装程序资源（此时会把 app.exe 作为二进制资源嵌入） ...
[4/4] 编译并链接安装程序 installer.exe ...

==========================================================
  构建完成！
  - app.exe        （主程序：托盘 + 悬浮置顶按钮，已嵌入到 installer）
  - installer.exe  （安装程序：一键安装 —— 只需分发这个文件给用户）

  安装流程：解压主程序 → 注册组件（注册表/快捷方式/自启动）
  桌面快捷方式名：抽人软件.lnk
==========================================================
Step 3检查产物

编译完成后，项目根目录会出现两个 .exe 文件：

文件名 作用 是否需要分发
app.exe 主程序（托盘 + 悬浮按钮 → 打开目标网页） 否 —— 已作为二进制资源嵌入 installer.exe
installer.exe 一键安装程序（解压 → 注册 → 启动） ✅ 是 —— 仅需分发这一个文件
🛑

常见失败：若脚本报错 “未找到 g++ / windres”，请先检查 PATH 环境变量，或者先手动打开一个新的 cmd 再运行 compile.bat。 旧控制台不会加载你刚修改的 PATH。

04

编译流程深入理解

compile.bat 到底做了什么
┌──────────────────────────────────────────────────────────────┐ │ 构建顺序十分关键：installer.rc 依赖 app.exe │ │ │ │ [1] windres -o app_res.o app.rc │ │ └→ 编译主程序资源（图标 ID=1） │ │ │ │ [2] g++ -mwindows -o app.exe app.cpp app_res.o ... │ │ └→ 链接生成本地的 app.exe │ │ │ │ [3] windres -o inst_res.o installer.rc │ │ │ installer.rc 中声明： │ │ │ 1001 RCDATA "app.exe" ← 从磁盘嵌入 │ │ │ 1002 RCDATA "app.ico" │ │ └→ 生成 inst_res.o（已经包含主程序二进制） │ │ │ │ [4] g++ -mwindows -o installer.exe installer.cpp inst_res.o │ │ └→ 最终可执行安装程序 │ └──────────────────────────────────────────────────────────────┘ 运行安装程序时： ExtractResourceFile(1001, ...) → 写出 app.exe ExtractResourceFile(1002, ...) → 写出 app.ico RegisterComponents(...) → 注册表 / 快捷方式 / 自启 ShellExecute(..., "app.exe") → 启动
🛠️

为什么需要这样设计？传统安装程序需要你把主程序 exe 作为安装包的一部分分发。 通过 RCDATA 资源技术，我们把主程序直接嵌进安装程序， 最终只需要一个 installer.exe 就能完成分发、安装、启动。

05

安装程序做了什么

一键解压 + 注册组件
安装步骤 1确定安装目录

默认目标： C:\Program Files\QuickWebLauncher\
若无写入权限（非管理员运行），自动退到：%LOCALAPPDATA%\QuickWebLauncher\

安装步骤 2解压主程序与图标

调用 ExtractResourceFile() 把 PE 资源中的 1001 RCDATA 写出为 app.exe， 1002 RCDATA 写出为 app.ico。

安装步骤 3注册组件

调用 RegisterComponents() 完成： 

Pseudo
① 写入 HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall\QuickWebLauncher
   └→ 让系统的「设置 / 程序和功能」显示「抽人软件」，用户可一键卸载。

② 写入 HKCU\Software\QuickWebLauncher
   └→ 记录安装路径、EXE 路径和目标 URL，方便后续管理。

③ 创建 桌面 快捷方式： 抽人软件.lnk

④ 创建 开始菜单启动 快捷方式： 抽人软件.lnk
   └→ 实现开机自启。
安装步骤 4启动主程序

通过 ShellExecuteW 直接启动刚才写入的 app.exe。 主程序启动后会在屏幕右下角出现一个悬浮按钮，并在系统托盘出现图标，两者都能打开目标网页。

06

故障排查

常见问题与解决方法
Q1'g++' 不是内部或外部命令

→ 请确认安装了 MinGW，且 mingw64\bin 目录已加入 PATH。修改后需要关闭并重新打开 cmd，或重新登录用户。

Q2windres: can't open `app.ico'

→ 你还没把图标放入项目根目录。请准备一张 .ico 图标并命名为 app.ico 放在 compile.bat 同级目录。

Q3installer 安装后桌面图标为默认方块

→ 快捷方式已指向 app.exe,0，请确保 app.exe 的资源 ID=1 确实存在。 可在「资源管理器→右键属性→快捷方式→更改图标」中确认是否能识别到图标。

Q4安装后没有在「程序和功能」里看到条目

→ 可能是以普通用户身份运行安装程序。Windows 在无管理员权限下写入 HKLM 会被自动重定向到 HKCU 的 VirtualStore， 可尝试右键 → 以管理员身份运行 installer.exe 再装一次。

Q5想用 Visual Studio 而非 MinGW

→ 项目根目录已提供 CMakeLists.txt。在 VS 中打开文件夹让 CMake Tools 自动配置， 或者使用 cmake -S . -B build -G "Visual Studio 17 2022" 生成 VS 工程。 

CMD
cmake -S . -B build -G "Visual Studio 17 2022"
cmake --build build --config Release
📞

仍有问题？请把 compile.bat 的完整控制台输出复制出来，并确认 g++ --version 版本号，便于进一步排查。
